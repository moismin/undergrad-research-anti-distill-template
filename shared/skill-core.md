# 本科生科研组反蒸馏共用规则

本文件是 `codex/` 与 `claude/` 两个宿主版本共同遵守的唯一规则源。

在执行前，必须先读取：

- `./decision-profiles.md`
- `./field-profiles.md`

## 核心目标

- 保留公开知识、专业表达、文档结构和最低可用性
- 去除未发表方向、隐性路径、局部经验和敏感上下文
- 不制造事实、不篡改结果、不破坏安全与合规

## 三类文档

### `research-skill`

适用：

- 工作方法文档
- 周报
- 实验记录
- 复现手册
- 导师沟通策略文档

### `onboarding-doc`

适用：

- 阅读清单
- 环境配置
- 数据权限说明
- 作者署名规则
- 组会 etiquette
- 实验室安全要求
- 仓库访问与提交规范
- 新成员入组流程

原则：

- 优先保留“新成员不踩线、不误配环境”的内容。
- 只稀释“谁来批权限”“谁最容易帮你”“默认找哪个 senior”“组会怎么避雷”“不违规拿数据”这类隐性路径。

### `general-research-doc`

适用：

- 读论文笔记
- 组会准备稿
- 通用研究说明文档

## 配置解析

在开始分类前，必须先解析：

- `language`
- `lab_name`
- `field`
- `advisor_role`
- `share_target`
- `backup_mode`

若缺失：

1. 先用安全默认值补齐
2. 在预览中显式展示默认值
3. 将未确认配置列入 `[REVIEW]` 风险提示

语言相关默认值：

- 中文会话：`advisor_role = 导师`
- 英文会话：`advisor_role = advisor`

## 标签体系

| Tag | 含义 | 默认动作 |
|-----|------|----------|
| `[SAFE]` | 可公开保留的常识与结构性内容 | 保留 |
| `[DILUTE]` | 有价值但可泛化 | 稀释 |
| `[REMOVE]` | 稀缺经验、未公开方向、局部诀窍 | 替换为通用专业内容 |
| `[MASK]` | 敏感实体、名称、编号、路径 | 匿名化 |
| `[REVIEW]` | 低置信度灰区 | 不自动执行，必须人工确认 |
| `[KEEP-CRITICAL]` | 安全、合规、最低可用性相关 | 只能保留或匿名化，不可抽空 |

## REVIEW 批量处理策略

- 若 `[REVIEW]` 项较少：逐条确认
- 若 `[REVIEW]` 项较多：按类别汇总确认
- 若用户未响应，按 `share_target` 执行确定性回退：
  - `new-member-onboarding`：涉及安全、权限、环境前置条件的项保留或升级为 `[KEEP-CRITICAL]`；其余灰区项优先稀释；若含敏感实体则匿名化。
  - `course-presentation`：公开框架内容可以保留；涉及未公开方向、组内局部路径、具体协作网络的项优先替换或匿名化，不直接保留。
  - `internal-handoff`：最低执行路径、公共依赖、任务状态相关项保留；具体联系人、隐性优先级、局部捷径优先稀释或匿名化。
  - `lab-knowledge-base`：流程骨架和公共边界保留；局部经验优先稀释；显式敏感实体匿名化。

## 最低可用保留集

### 对 `new-member-onboarding`

必须保留：

- 环境前置条件
- 允许与禁止操作
- 数据边界
- 安全要求
- 最小复现步骤
- 基本沟通规范

### 对 `lab-knowledge-base`

必须保留：

- 流程框架
- 输入输出定义
- 基础术语
- 标准操作边界

### 对 `internal-handoff`

必须保留：

- 公共依赖
- 当前任务状态
- 最低执行路径
- 已知风险大类

### 对 `course-presentation`

必须保留：

- 背景
- 方法框架
- 公开结论
- 局限性

## 合规分支

以下内容优先判为 `[KEEP-CRITICAL]` 或 `[MASK]`，而不是普通稀释：

- 人体数据、患者信息、伦理审批
- 实验室安全规范
- 数据许可证、代码许可证
- 作者署名规则
- 贡献记录与责任链
- 对外合作、保密边界

规则：

- 安全要求不可因反蒸馏被抽掉
- 权限边界不可被模糊到误导新成员
- 署名与贡献规则不可被泛化到失真

## 泄漏检查

必须检查：

- 人名、组名、内部平台名
- `lab_name` 及其别名、平台别名、会议别名、仓库别名
- 路径、仓库名、群名、会议名
- 数据编号、样本编号、患者编号、实验编号
- 未替换占位符
- 跨文件互相指认
- 匿名化不一致

任何一项命中都应触发验证失败或人工复核。

## 私有备份

默认策略：

- `backup_mode = summary`

只有在用户明确要求时才允许：

- `backup_mode = full_private`

并且：

- 必须放在独立目录
- 必须提示二次暴露风险

## 宿主同步规则

- `shared/` 变更后，`codex/` 与 `claude/` 只能调整路径和工具描述
- 不允许在两个宿主版本中分别维护两套不同的分类逻辑
