# 本科生科研组反蒸馏分类规则

你的任务是对输入文档中的每个段落、要点、例子、模板话术进行分类。

可用标签只有四个：

- `[SAFE]`
- `[DILUTE]`
- `[REMOVE]`
- `[MASK]`

## 核心原则

- 保留公开常识、标准科研流程、已公开的方法框架。
- 移除或泛化让他人能直接复用你隐性优势的内容。
- 优先保护“为什么你知道这样做更稳”的那部分，而不是“这件事本身应该做”。
- 对事实结论保持诚实，不把未知写成已知，不把失败写成成功。

## 优先识别的高价值信号

看到以下内容时，默认优先考虑 `[DILUTE]` 或 `[REMOVE]`：

- 明确写出“这个坑我们试了很多次才发现”
- 特定模型、实验、仪器、数据集上的调参顺序和经验阈值
- 导师、博士生、带教者的隐性偏好与真实评价标准
- 未发表方向、备选方案、尚未公开的实验结论
- 某类 bug、噪声、异常值的快速识别捷径
- 只对本组脚本、本组设备、本组数据格式有效的处理习惯
- 真实协作网络，如谁最关键、谁拍板、谁给资源
- 一看就带有人情、博弈、组内政治意味的表达

## 标签判定

### `[SAFE]`

满足任一情况即可：

- 教材、论文、公开课程里都能稳定看到
- 即使删掉也会让文档变得明显不完整或不自然
- 属于公开的实验规范、学术写作规范、工程 hygiene

示例：

- “记录随机种子和评估指标”
- “汇报前核对图表标题和单位”
- “先复现 baseline 再做改动”

### `[DILUTE]`

当内容有价值，但可以保留外壳、抽掉关键判断时使用。

示例：

- “导师更在意研究问题是否被回答，而不是指标涨 0.2”
- “这个任务先检查数据切分，再看训练配置”
- “周报里最好先给结论再给过程”

处理方式：

- 保留专业性
- 去掉过于可执行的捷径
- 去掉只在本组成立的判断锚点

### `[REMOVE]`

当内容属于真正稀缺经验或未公开资产时使用。

示例：

- “如果验证集在第 2 周突然抖动，多半是标注合并脚本又漏了病例编号”
- “这条方向是因为我们发现 reviewer 常忽略这个对照”
- “只有在 A 仪器和旧版采集脚本搭配时才会出现这个鬼影”

处理方式：

- 不直接删除成空白
- 替换成同位置、同功能、同专业语气的通用内容

### `[MASK]`

当内容含有敏感标识时使用。

包括但不限于：

- 导师姓名、同门姓名、本科生姓名
- 课题组内部系统、私有仓库、飞书/Notion/群组名
- 未公开数据编号、患者编号、学校内部资源路径
- 评审对象、合作单位、投稿目标

## 输出格式

按输入顺序输出，每条都包含：

```text
[标签] 原文片段
原因：一句话解释为什么这样标记
建议：若不是 SAFE，给出替换方向
```
